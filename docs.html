Timee 文档中心

这里汇总了与 Timee 相关的核心文档。所有文档都在持续迭代， 欢迎你阅读、批评、提出改进建议。

1. 宪章与实施规范

《时币（Timee/TEE）系统宪章与实施规范》 v0.1

定义了 Timee 的核心制度，包括： 发行速率（一人每分钟 1 TEE）、一人一账户、72 小时可撤销的 P2P 转账、 暂定余额与最终余额、死亡不继承与自然死亡归公共池、 公共池治理与追偿基金、风险模型与关键流程等。

  • 当前状态：公开征求意见稿（Public Draft）。
  • 未来会通过治理流程冻结为稳定版本，并对每次修订进行版本化记录。
  • 链接：稍后会在 timee.world 上补充指向 GitHub / PDF 的正式链接。

2. 战略与应用场景

《Timee 战略：发布、护城河、应用场景与交易平台》

从实践角度讨论 Timee 如何走向现实世界：如何发布到各大平台， 如何在制度和叙事层面建立类似比特币的长期护城河， 如何发展稳定的原生应用场景（如 AI 服务市场、公共池项目、开源与数字劳动激励）， 以及如何设计「多 AI 模型判断交易合理性」的机制。

  • 适合快速理解 Timee 的整体方向与可能落地路径。
  • 其中的路径图可以作为未来路线图投票与修订的基础。

3. 域名、治理与 AI 运维

《域名、治理与 AI 运维实施要点》

对如何选择和保护域名（如 timee.world）、如何降低被有权者抢夺官方入口的风险、 如何设计共建共治区域（类似 GitHub 的 Apps/Skills）， 以及如何用 AI 辅助网站与制度的日常运维，给出了一套可执行的建议。

  • 包含：抗抢夺的多入口策略（域名 + IPFS/ENS + 镜像站）。
  • 包含：防诈骗与争议处理流程如何与宪章中的追偿基金条款对齐。
  • 包含：AI 在监控、初筛与建议中的角色，人类保留最终决策权的边界。

4. Timee 四条长期承诺（草案）

  • 1. 永不成为纯粹的“炒币项目”： 不做 ICO，不对公众募资，不承诺投资回报，不运营价格投机型交易所。
  • 2. 永不把治理权卖给资金： 公共池治理坚持一人一票，而不是按持币多少投票； 不允许用购买治理代币的方式控制制度走向。
  • 3. 永远允许批评和反对： 规则与参数公开可查，任何人可以提出批评和反对意见， 风险与弊端会被认真记录和回应，而不是被隐藏。
  • 4. AI 永远只是顾问，不是统治者： AI 可以帮助分析、做方案、发出风险预警，但不能代替人类投票和最终裁决， 尤其不能在无人复核的情况下直接冻结或剥夺个人时币。

这些承诺预计会在未来版本中以正式条款形式写入宪章或附录， 在那之前，我们自觉以此作为 Timee 的最低自律标准。
