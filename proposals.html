Timee 提案模板：把想法变成可讨论文本

本页只教你“怎么写提案”，不重复讲文明宣言与制度细节。想看制度：去 宪章。想贡献资源：去 Commons。

1) 提案适用范围

  • 公共池（PP）资金如何使用（项目、基础包、公共服务等）。
  • 规则或参数的修改建议（风控阈值、追偿规则细节等）。
  • 应用场景（AI 服务、工作悬赏、线下试点等）。
  • 风险与滥用方式的预警，以及防范建议。

短评或简单提问建议放在 FAQ 或 GitHub Issues 中。

2) 标准提案结构（建议按这个写）

（1）标题与基本信息

  • 提案标题：一句话概括你想做什么。
  • 提案人：意向账户名或昵称。
  • 提案类型：公共池项目 / 基础包 / 规则修改 / 风控与反诈 / 应用场景 / 其他。

（2）背景与问题

  • 现实中你观察到的问题是什么？（越具体越好）
  • 影响到哪些人？规模大概多大？
  • 如果不做，最可能发生什么？

（3）你的方案

  • 你希望 Timee 做什么？（动作 + 对象 + 边界）
  • 若涉及公共池：预算多少、分几期、验收方式。
  • 若是规则修改：当前文字是什么、建议改成什么、为什么。

（4）影响分析

  • 主要受益人是谁？
  • 短期/长期各有什么变化？
  • 可能伤害到哪些人？（哪怕少数也要写）

（5）风险与失败场景（最重要）

  • 最担心如何被滥用？（骗补、洗钱、操纵、歧视等）
  • 失败时会怎样？有没有刹车条件？

（6）验收与时间表

  • 什么算成功？用什么指标或证据验证？
  • 大致时间线：启动/中期复盘/最终评估。

（7）与宪章硬规则的兼容性

  • 确认不违反：一人一账户、个人不可剥夺、公共池一人一票、AI 仅辅助人类决策等。

3) 当前提交方式（用 GitHub Issues）

  1. 打开 Timee 仓库 Issues
  2. 点击 New issue，标题建议：[提案] 简短标题
  3. 正文按上面结构填写（写不全也没关系，先把背景/方案/风险写出来）。

未来有专门投票与提案系统时，这些 Issue 可以迁移并保留历史记录。
