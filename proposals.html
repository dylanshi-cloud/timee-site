Timee 提案模板

不需要会写制度文本，你只要按下面的提纲，把你的想法解释清楚，就已经是很好的提案了。 未来正式治理上线后，这些提案可以作为投票和讨论的起点。

1. 提案适用范围

你可以为这些主题写提案：

  • 公共池（PP）资金如何使用（项目、基础包、公共服务等）。
  • 规则或参数的修改建议（例如风控阈值、追偿规则的细节等）。
  • 新的应用场景（如 AI 服务市场、工作悬赏平台、线下试点）。
  • 对潜在风险或滥用方式的警告，以及防范建议。

短评或简单问题可以直接去 FAQ 或 Issues，这个模板主要面向“需要系统讨论”的想法。

2. 标准提案结构（建议格式）

（1）标题与基本信息

  • 提案标题：一句话概括你想做什么。
  • 提案人：可以用意向账户名或昵称。
  • 提案类型：从以下中选一项或多项：
    • 公共池项目 / 基础包 / 规则修改 / 风控与反诈 / 应用场景 / 其他

（2）背景与问题

  • 你观察到的现实问题是什么？（越具体越好）
  • 这个问题影响到了哪些人？（例如失业者、照护者、老年人等）
  • 如果什么都不做，最可能出现什么后果？

（3）你的建议方案

  • 你希望 Timee 做什么？（清楚写出动作和对象）
  • 如果涉及公共池，请说明：
    • 预算大致是多少 TEE？
    • 分几期拨付？（例如启动费 / 中期检查 / 验收）
  • 如果是规则修改，请说明：
    • 你希望修改哪一条规则或参数？
    • 当前文字是什么？你建议改成什么？

（4）受益人与影响分析

  • 主要受益人群是谁？他们大概有多大规模？
  • 短期内有什么直接好处？长期可能带来什么结构性变化？
  • 是否会对某些人产生负面影响？（哪怕是少数，也请写出来）

（5）风险与失败场景

  • 你最担心这个提案会如何被滥用？（例如被某些群体占便宜、骗补、洗钱等）
  • 如果项目失败或效果不好，会发生什么？
  • 有没有“最坏情况”，以及可以提前设定的刹车条件？

（6）验收标准与时间表

  • 如何判断这个提案“算是成功了”？（尽量用可量化或可观察的指标）
  • 大致时间线（例如 3 个月启动、6 个月评估、12 个月复盘）。

（7）与宪章硬规则的兼容性

  • 确认你的提案不会违反以下硬规则：
    • 一人一账户；
    • 个人时币不得被任意剥夺或强制征收；
    • 公共池一人一票、透明可审计；
    • AI 只做顾问，人类最终决策。
  • 如果有潜在冲突，请明确指出并解释你的看法。

3. 在 GitHub 上提交提案的步骤（当前方式）

  1. 打开 Timee 仓库的 Issues 页面 。
  2. 点击「New issue」，标题建议采用格式： [提案] <简短标题>
  3. 在正文中按照上面的提案结构逐条填写。如果一时写不全，也可以先写“背景 + 你的建议 + 你最担心的风险”。
  4. 提交后，你可以随时补充、修改和回复他人的评论。

未来如果有专门的提案系统和投票界面，这些 GitHub 提案可以迁移过去， 并作为未来讨论和投票的历史记录。

4. 写提案的一些小建议

  • 尽量用具体场景和数字来说明问题，而不是抽象口号。
  • 不要害怕提出“激进”想法，只要你同时认真思考了风险和失败场景。
  • 欢迎你提出对 Timee 的批评性提案，比如：
    • “这样的设计会不会让某些人被排除在外？”
    • “会不会无意中制造歧视或不公？”
  • 如果你不会用这个模板，也可以只写一段自然语言说明， 未来可以交给 AI 或其他志愿者帮忙整理成正式格式。
